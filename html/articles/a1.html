2. 解决因子元素设置float: left导致父元素（如果未设置高度）高度为0

css代码为：

                    .clr::before,
                    .clr::after
                    {
                        content: "";
                        display: block;
                        clear: both;
                        height: 0;
                        overflow: hidden;
                        visibility: hidden;
                    }

给需要清除浮动的元素加上类.clr即可


3. 解决元素内部第一个子元素设置margin-top导致该元素应用设置问题

在给一个元素的第一个子元素设置margin-top值时，会发现是父元素应用了该样式，好像这句设置发生了转移一样。如下图所示：

                    样式代码如下：
                    .group
                    {
                        width: 500px;
                        height: 600px;
                        border: 1px
                        solid #000;
                    }
                    .parent
                    {
                        width: 100%;
                        height: 400px;
                        background-color: #f00;
                    }
                    .child
                    {
                        width: 100%;
                        height: 200px;
                        background-color: #0f0;
                        margin-top: 200px;
                    }

原本是想让绿色的子元素距离红色的父元素顶部200px，结果这个样式却是父元素应用了。

解决方案如下：

给父元素加上样式"overflow: hidden"即可。

这样就可以正常显示样式了。
